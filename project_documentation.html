Complete End-to-End Project Documentation

1. Project Overview

ViralMind AI is an AI-powered content generation pipeline. It transforms raw meeting summaries or CEO insights into highly engaging, production-ready social media content (tweets, topics, and video scripts). It utilizes a 6-step sequential pipeline with real-time web search capabilities to align AI generation with live industry trends.

2. Architecture & Tech Stack

Core Languages & Frameworks

  • Next.js 16.1.7 (App Router): React framework used for robust server-side routing, API routes, and optimized rendering.
  • React 19.2.3: For building highly interactive UI components.
  • TypeScript: Provides strict static typing across the entire pipeline.
  • Tailwind CSS v4: Utility-first CSS framework for rapid, responsive UI styling using a custom dark theme with neon yellow/green accents.

Backend, APIs & State

  • Groq AI (llama-3.3-70b-versatile): The core inference engine used for blazing-fast text generation.
  • Tavily Search API: Used to fetch live internet context to ground the AI responses.
  • Firebase 12.10.0: Configured for future-proof Cloud Firestore database scaling and User Authentication.
  • Zustand: A lightweight, highly scalable state management library used to control the 6-step pipeline state without React Context overhead.

3. Detailed Folder & File Structure

app/ (Next.js App Router)

  • page.tsx

    The landing page. Contains dynamic background hero animations and CTA button to route to /dashboard.

  • layout.tsx & globals.css

    Defines the root HTML shell, font configurations, and global Tailwind CSS imports.

  • dashboard/page.tsx

    The core application wrapper. It watches the Zustand store's `currentStep` and dynamically renders the appropriate pipeline component (Step 1 to 6).

app/api/ (Backend Route Handlers)

api/search/route.ts

Hits Tavily API via POST. Validates query, fetches 5 results, and formats them into a 2000-character context block.

api/analyze/route.ts

Step 2: Merges meeting text + Tavily context => Outputs 4 Problems, 4 Solutions, 3 Trends using `ANALYZE_PROMPT`.

api/tweets/route.ts

Step 3: Takes analyzed data => Generates 6 highly specific Tweets in different viral formats using `TWEETS_PROMPT`.

api/topics/route.ts

Step 4: Suggests 8 video reel/carousel topics based on the tweets and problems using `TOPICS_PROMPT`.

api/scripts/route.ts

Step 5: Takes the selected topics => Writes dual scripts (CEO & Influencer styles) using `SCRIPTS_PROMPT`.

components/pipeline/ (UI Modular Views)

  • SummaryInput.tsx: Form for CEO Name, target audience, and pasting the meeting summary. Triggers Tavily search simultaneously with submission.
  • ProblemSolution.tsx: Displays the parsed JSON analysis (Problems, Solutions, Trends).
  • TweetGenerator.tsx: Visual grid showing the generated tweets with copy buttons.
  • TopicSelector.tsx: A selectable grid of 8 video ideas. Users must select up to 5 topics. Updates Zustand array.
  • ScriptViewer.tsx: A dual-tab interface showing the generated scripts (Business tone vs Influencer tone).
  • StepIndicator.tsx: The visual progress bar across the top of the dashboard.

lib/ & store/ & types/ (Core Logic)

  • lib/groq.ts: Initializes Groq SDK. Exports a `chatCompletion` wrapper ensuring JSON-formatted LLM responses.
  • lib/prompts.ts: The absolute brain of the app. Contains highly engineered TypeScript prompt functions ensuring rigid JSON structure outputs.
  • lib/firebase.ts: Prepares the app for database interactions (saving sessions, user auth).
  • store/pipeline.ts: The Zustand central store. Holds all data (`profile, analysis, tweets, topics`) so components don't need messy prop-drilling.
  • types/index.ts: All custom TypeScript interfaces (`CEOProfile, AnalysisOutput, Tweet, etc`).

4. Core Implementation Deep Dive

How AI Responses are Enforced (JSON Mode)

Instead of regex parsing natural language, the app forces Groq/Llama-3.3 to return strict JSON arrays.

// lib/groq.ts lines 7-22
export async function chatCompletion(prompt: string, maxTokens: number = 2048): Promise<string> {
  const completion = await groq.chat.completions.create({
    model: 'llama-3.3-70b-versatile',
    messages: [{ role: 'user', content: prompt }],
    max_tokens: maxTokens,
    temperature: 0.7,
    response_format: { type: 'json_object' }, // Critical: Forces standard JSON output
  })
  return completion.choices[0]?.message?.content || ''
}

// app/api/analyze/route.ts lines 10-13
const text = await chatCompletion(prompt, 1024)
// Strip markdown codeblocks (Groq sometimes adds ```json wrappers)
const parsed = JSON.parse(text.replace(/```json|```/g, '').trim())
return NextResponse.json(parsed)

Zustand State Management Flow

Instead of passing props between 6 nested components, any component can dispatch an action to store.

// store/pipeline.ts lines 32-58
export const usePipelineStore = create<PipelineStore>((set) => ({
  ...initialState,
  
  setStep: (step) => set({ currentStep: step }),
  
  toggleTopic: (index) =>
    set((state) => {
      const selected = [...state.selectedTopics]
      const i = selected.indexOf(index)
      if (i === -1 && selected.length < 5) selected.push(index) // Max 5 selection logic
      else if (i !== -1) selected.splice(i, 1) // Deselection logic
      return { selectedTopics: selected }
    }),
}))
